# User Personas — 테이블오더 서비스

본 문서는 테이블오더 서비스의 주요 사용자 유형을 정의합니다. 기본 수준(역할/목표/Pain Points/주요 행동) 포맷을 사용합니다.

---

## Persona 1: 매장 고객 (Diner)

| 항목 | 내용 |
|---|---|
| **이름/별명** | "Diner Dave" |
| **역할** | 매장에 방문하여 테이블에서 음식을 주문하는 일반 고객 |
| **사용 기기** | 테이블에 설치된 태블릿 (브라우저 기반 웹 UI) |
| **기술 숙련도** | 낮음~중간 (스마트폰 앱 사용 경험 수준) |

### Goals (목표)
- 점원 호출 없이 바로 메뉴를 보고 빠르게 주문하기
- 현재 내 테이블 주문 상태 (대기중/준비중/완료) 확인
- 추가 주문을 쉽게 넣기

### Pain Points
- 점원을 부르기 어렵거나 대기 시간이 긴 상황
- 메뉴 설명/이미지를 못 봐서 어떤 메뉴인지 모를 때
- 주문 실수(잘못 담음)를 쉽게 수정하고 싶음
- 내가 주문한 게 제대로 접수됐는지 불안

### 주요 행동 (Key Behaviors)
- 착석 후 태블릿 화면의 메뉴 카테고리를 훑어본다
- 메뉴 상세(이미지/설명/가격)를 확인하고 장바구니에 담는다
- 장바구니에서 수량/삭제로 조정하고 주문 확정
- 주문 후 메뉴 화면으로 자동 복귀하면 추가 주문을 한다
- 주문 내역 페이지에서 상태를 확인한다

### 접근 가능한 기능
- 메뉴 조회, 장바구니, 주문 생성, 현재 세션 주문 조회

## Persona 2: 매장 관리자 (Store Admin)

| 항목 | 내용 |
|---|---|
| **이름/별명** | "Manager Mia" |
| **역할** | 매장을 운영하며 주문 접수/조리 진행/테이블 관리를 담당 |
| **사용 기기** | 카운터 PC 또는 관리자용 태블릿 (관리자 웹 UI) |
| **기술 숙련도** | 중간 (POS/관리 프로그램 사용 경험) |

### Goals (목표)
- 신규 주문을 즉시 인지하고 조리 상태를 업데이트
- 여러 테이블 상황을 한눈에 파악
- 실수 주문을 빠르게 삭제(직권 수정)
- 테이블 이용 완료 시 깔끔하게 세션을 닫기
- 메뉴 정보를 쉽게 업데이트

### Pain Points
- 주문 누락/지연 인지 실패 시 고객 컴플레인
- 테이블별 현재 주문 상태/총액 파악이 어려움
- 이전 테이블 주문이 새 고객 화면에 남아 있는 혼란
- 메뉴 변경 시 시스템 수정이 번거로움

### 주요 행동 (Key Behaviors)
- 매장 식별자+아이디/비밀번호로 관리자 UI 로그인 (16h 세션)
- 실시간 대시보드에서 테이블별 카드를 모니터링 (신규 주문 강조)
- 주문 카드 클릭 → 상세 확인 → 상태 전환 (대기중 → 준비중 → 완료)
- 잘못된 주문은 직권 삭제 (확인 팝업 후)
- 테이블 이용 완료 처리 → 현재 주문 초기화 + 이력으로 이동
- 메뉴 등록/수정/삭제, 노출 순서 조정
- 새 테이블 태블릿 초기 설정 수행

### 접근 가능한 기능
- 관리자 인증, 실시간 주문 모니터링 (SSE), 테이블 관리, 메뉴 관리, 주문 상태 변경/삭제, 과거 이력 조회
